# Validation Methods

Build the smallest artifact that tests the riskiest assumption. Minimize cycle time.

## MVP Selection

| Uncertainty Type | MVP Options | Effort |
|------------------|-------------|--------|
| Demand | Landing Page, Video/Demo | Low |
| Solution | Concierge, Wizard of Oz | Medium |
| Execution | Piecemeal, Single Feature | Medium-High |

| MVP Type | Purpose |
|----------|---------|
| Landing Page | Test value proposition with signup/waitlist |
| Video/Demo | Explain complex concept before building |
| Concierge | Deliver service manually to validate solution |
| Wizard of Oz | Simulate automation with human backend |
| Piecemeal | Combine existing tools for quick market test |
| Single Feature | Build one feature to test core hypothesis |

## Quantitative Validation

| Test | Method | Pass Threshold |
|------|--------|----------------|
| Landing Page | Page + CTA → targeted ads → measure conversion | ≥10% signup |
| Fake Door | Add button for non-existent feature → measure CTR | ≥3x baseline CTR |
| Pre-order | Request payment before building | ≥5% conversion |
| Sean Ellis | Survey: "How disappointed without this product?" | ≥40% "very disappointed" |

### Sean Ellis Survey Options
- Very disappointed
- Somewhat disappointed
- Not disappointed

## Qualitative Validation

### Problem Interviews

Confirm problem exists and matters. Sample size: 5-10 interviews.

**Questions:**
1. Describe the last time you experienced [problem]
2. How often does this occur?
3. What solutions have you tried?
4. What is hardest about [problem area]?
5. If you had a magic wand, what would you change?

| Signal | Indicators |
|--------|------------|
| Red flag | Vague answers, rare occurrence, no solution attempts, low emotion |
| Green flag | Specific stories, high frequency, multiple attempts, strong emotion |

### Solution Interviews

Confirm solution resonates.

**Questions:**
1. [Show concept] What is this for?
2. Would this solve the problem we discussed?
3. What would make this more useful?
4. What concerns would you have?
5. What would you pay for this?

## Jobs-to-be-Done Analysis

### JTBD Statement Format

`When [situation], I want to [motivation], so I can [outcome].`

Example: "When onboarding a new team member, I want to quickly share relevant documentation, so I can get them productive without repeated explanations."

### Job Dimensions

| Dimension | Question |
|-----------|----------|
| Functional | What task are they accomplishing? |
| Emotional | How do they want to feel? |
| Social | How do they want to be perceived? |

### Competing Solutions

Map all ways customers "hire" solutions: direct competitors, indirect alternatives, DIY workarounds, do nothing (often the largest competitor).

### Switching Triggers

| Factor | Effect |
|--------|--------|
| Push | Problems with current solution drive change |
| Pull | Attraction to new solution motivates switch |
| Anxiety | Fears about switching create resistance |
| Habit | Comfort with status quo maintains inertia |
